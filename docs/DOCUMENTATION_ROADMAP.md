# DOCUMENTATION_ROADMAP – Capsule

## Ziel

Die Projektdokumentation soll nicht nur „vorhanden“, sondern für Betrieb, Weiterentwicklung und Handoff tatsächlich nutzbar sein.

## 2. Unmittelbar priorisierte Doku-Arbeit

### Priorität A
- `PROJECT_STATE` auf aktuellen Stand halten
- `RUNBOOK` vollständig und praxisnah halten
- Handoff-Standard stabilisieren

### Priorität B
- Deployment-Zielbild explizit dokumentieren
- VPS-Runbook ausarbeiten und nach echtem Deploy schärfen

### Priorität C
- Manifest projektbezogen ergänzen
- Dokumentierte Restschulden und Betriebsregeln explizit machen

## 3. Regel für künftige Pflege

Nach jedem relevanten Meilenstein prüfen:

- Hat sich Architektur geändert?
- Hat sich Deploy / Runtime geändert?
- Hat sich der Handoff-Einstieg geändert?
- Hat sich das Betriebsmodell Custom GPT / Website verändert?

Wenn ja, dann Doku im selben Arbeitsblock aktualisieren.
